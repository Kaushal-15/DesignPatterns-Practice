# DesignPatterns-Practice

A collection of design patterns implemented in C++ as part of my journey to understand software design principles and write better object-oriented code.

The goal of this repository is not just to memorize design patterns, but to understand:

- Why a pattern exists
- What problem it solves
- When to use it
- When not to use it
- Real-world applications

Each pattern contains a simple implementation, explanation, and examples to make the concepts easier to understand.

---

## Repository Structure

```text
DesignPatterns-Practice
│
├── behavioral
├── creational
└── structural
```

Patterns are organized according to the GoF (Gang of Four) classification.

## Why This Repository?

While preparing for system design interviews and improving my understanding of object-oriented design, I realized that knowing the theory isn't enough.

Implementing patterns from scratch helps in understanding:

- SOLID Principles
- Low Coupling
- High Cohesion
- Maintainable Code
- Extensible Architectures

This repository serves as my personal reference and learning journal.
